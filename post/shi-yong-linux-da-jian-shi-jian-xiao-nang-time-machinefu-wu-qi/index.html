使用 Linux - deepin 搭建 Apple 时间机器（Time Machine）服务器

发布于 2019-07-07 | 4分钟 | 945字数

由于前段时间mac屏幕突然报废，意识到了数据备份的重要性，时常插硬盘等待备份有时候很是麻烦，所以正好实验室有一台自己用的装了deepin的服务器那就在上面搭个时间胶囊的服务器，连进校园网后能让他自动备份。
Netatalk v2

samba

在deepin上搭建一个Time Machine服务器，仍有部分问题需要解决

Netatalk v2

在下一个博客中该用Netatalk v3

问题：

  • 只能用ipv4

1.安装需要的软件

sudo apt install netatalk

一个afp服务器软件，其中包含avahi

2.准备文件夹

在想要备份的文件夹内新建TimeMachineFolder文件夹

3.设置netatalk

首先备份配置文件

sudo mv /etc/netatalk/AppleVolumes.default  /etc/netatalk/AppleVolumes.default.old

然后创建新配置文件

sudo touch /etc/netatalk/AppleVolumes.default

使用Vim在文件末尾添加下列语句

:DEFAULT: options:upriv,usedots
/home/tm/TimeMachineFolder "My Time Machine" options:tm volsizelimit:102400

注意将第2行的文件夹路径设定为你再第2步中创建的文件夹的路径。

第二行中的volsizelimit设定了Time Machine将会使用的最大硬盘空间，单位是MB。

4.重启netatalk

sudo service netatalk restart

5.在Mac上连接到Time Machine

首先直接尝试在Time Machine中选择这个服务器（会显示在可用磁盘下面，名字显示为第四步中你netatalk设置文件中指定的名字）。

如果你无法找到，那么打开Finder并按下⌘+K，在弹出来的窗口中，于服务器地址一栏输入

afp://IP.of.your.server/

然后点连接。

deepin里面自带samba

（Samba 提供文件共享、 Avahi) 提供服务广播的方式，搭建通过网络进行备份的 Time Machine 备份服务器，实现和 Time Capsule 体验一致的无干预自动备份。

Samba 最初是基于 Microsoft 的 SMB 文件共享协议开发的开源实现，Avahi 则是以 Apple 开源的 Bonjour 源码上发展而来的 zeroconf 实现，说到底还是原本就有的协议了。

1.配置 Samba 提供文件共享服务，让 macOS 能够访问到备份数据路径。

首先，添加一个 Time Machine 专用的用户。

我这里使用了个人用户，反正都是个人数据。

useradd dazzy

然后，设置该用户的 Samba 协议密码，这个密码跟 Linux 系统用户密码是相互独立的，需要单独设置。

smbpasswd -a dazzy

接着，就是配置 Samba 让它把备份数据路径暴露出去。

vim /etc/samba/smb.conf
#文件最后插入

#服务名称，会显示在 Time Machine 备份界面
[Time Machine]
#备份数据路径
path = /data/timemachine
#允许SMB客户端写入
writable = yes 
#允许访问的用户列表，避免备份数据泄漏
valid users = dazzy
#catia: 兼容linux/macos中合法但在Windows/SMB中非法的字符
#fruit: 增强macos的的兼容性，并提供Netatalk AFP服务
#streams_xattr: 通过Linux文件系统提供Windows NTFS属性字段
vfs objects = catia fruit streams_xattr
#启用Apple SMB客户端兼容
fruit:aapl = yes
#修改一下Finder里的图标
fruit:model = MacPro
#重要！开启 Time Machine 支持
fruit:time machine = yes
spotlight = yes

最后启动samba服务

systemctl enable --now smb

2.配置 Avahi

Avahi 不需要配置。没错，直接启动就好了。

systemctl enable --now avahi-daemon
