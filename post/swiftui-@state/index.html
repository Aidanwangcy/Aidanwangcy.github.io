SwiftUI: @State 一个用来修改页面状态的神奇语句

发布于 2020-01-22 | 分类于 Swift | 3分钟 | 541字数

在我现在接触到的 SwiftUI 开发中需要更改，保存页面的值的时候，比如按一个按钮就能增加一个值，当退出的时候这个值如何才能保存起来，在下次打开的时候还能使用就要用到 @State 这个属性装饰器。

那状态怎么改变？

比如写个按钮

struct ContentView: View {
    var count = 0
    var body: some View {
        Button("count: \(count)") {
            self.count += 1
        }
    }
}

当点击按钮的时候count加1。但是这个其实是会报错的 struct 是不可变的

这怎么办呢？
就可以用到@State

struct ContentView: View {
    @State var count = 0
    var body: some View {
        Button("count: \(count)") {
            self.count += 1
        }
    }
}

@State使得超出struct本身的限制，动态修改属性的值。这样就能在程序发生变化的时候，更新界面了。

双向绑定

还有个问题就是如果输入框的输入内容改变了，存储输入内容的属性应该如何更新？
这就需要讲到双向绑定了。当页面的UI发生变化的时候，对应的属性也能随之改变，真正做到页面时状态的函数。
在 SwiftUI 中使用 $来表示，在属性前面加上$就意味着读取属性的同时只要值改变也会更新属性的值
举个🍍：

struct ContentView: View {
    @State private var name = ""
    var body: some View {
        Form {
            TextField("Your name", text: $name)
            Text("Hello \(name)")
        }
    }
}

当你输入值的时候name也会随之变化
还可以给绑定属性添加动画包装，这样在属性值发生变化时页面的变动会有动画效果，比如下面的🍎，开关打开时，TextField会更平滑的出现：

Toggle(isOn: $addLoyaltyDetails.animation()) {
    Text("Add iDine loyalty card")
}
if addLoyaltyDetails {
    TextField("Enter your iDine ID", text: $loyaltyNumber)
}

自定义双向绑定

举个🥭自定义get set

struct ContentView: View {
    @State private var username = ""
    var body: some View {
        let binding = Binding(
            get: { self.username },
            set: { self.username = $0 }
        )
        return VStack {
            TextField("Enter your name", text: binding)
        }
    }
}

好处就是可以在自定义绑定的set和get中添加自己的逻辑，比如对数据的处理，或者先进行额外的操作，再进行set和get
